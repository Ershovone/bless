# /add-task — добавить задачу на доску

## Что делаешь
Создаёшь файл `.claude/tasks/<priority>-<slug>.md` по формату из `.claude/tasks/README.md`.

## Шаги

1. **Распарси ввод пользователя**. Если он дал только короткую формулировку — задай 1-2 уточняющих вопроса (через AskUserQuestion если есть, иначе обычный текст):
   - Какой `priority` (1-5)?
   - Какой `type` (refactor/bugfix/feature/chore/test/docs)?
   - Если непонятна Definition of Done — спроси «как поймём, что готово?»

2. **Сгенерируй slug** из заголовка: латиница, kebab-case, до 40 символов. Например «Вынести цвета карты в константы» → `extract-map-colors-constants`.

3. **Сгенерируй имя файла**: `<priority>-<slug>.md`. Если уже занято — добавь `-2`, `-3` и т.д.

4. **Запиши файл** по шаблону из README. Заполни:
   - `id: <slug>`
   - `status: open`
   - `priority: <1-5>`
   - `type: <type>`
   - `created: <сегодняшняя дата YYYY-MM-DD>`
   - `attempts: 0`
   - Тело: «Контекст», «Что нужно сделать» (опиши с ссылками на конкретные файлы, если знаешь), «Definition of Done» (чеклист), «Не трогать» (если применимо).

5. **Сообщи**: «Добавлено: `<имя файла>`. Сейчас в open: <N>».

## Важно
- НЕ создавай задачу, если описание слишком расплывчатое и пользователь не отвечает на уточнения. Лучше явно сказать «нужны детали, не сохраняю».
- НЕ дублируй: перед созданием просканируй существующие задачи, если найдёшь похожую — спроси «обновить эту?» вместо новой.
- DoD должна быть проверяемой. Не «улучшить UX», а «при клике на маркер открывается popup с заголовком и описанием».
