# Доска задач для агентов

Каждая задача — отдельный `.md` файл в этой папке. Имя = `<priority>-<slug>.md` (приоритет в имени даёт сортировку).

## Формат

```markdown
---
id: <slug, совпадает с именем без приоритета>
status: open | in_progress | done | blocked
priority: 1-5  # 1 = самый срочный
type: refactor | bugfix | feature | chore | test | docs
created: YYYY-MM-DD
attempts: 0
---

# Заголовок задачи

## Контекст
Зачем это делается, что за проблема.

## Что нужно сделать
Конкретные шаги/файлы, насколько известно.

## Definition of Done
- [ ] условие 1
- [ ] условие 2

## Не трогать
Файлы/области вне скоупа.
```

## Правила для агентов

- **open** → агент берёт, ставит `in_progress`, инкрементит `attempts`
- При успехе → `done` + создаёт PR со ссылкой `Closes task: <id>` в описании
- При провале (билд/тип/тест упали и не смог починить за разумное время) → возвращает `open` с заметкой `## Заметки от попытки N` в теле, не больше 3 attempts — после блок `blocked`
- Никогда не пушить в `main`, всегда отдельная ветка `task/<id>`
- Перед PR обязательно: `npm run lint`, `npx tsc --noEmit`, `npm run build`

## Приоритет
1 — критика/баги, 2 — UX-полировка, 3 — рефакторинг, 4 — оптимизация, 5 — мелочь
